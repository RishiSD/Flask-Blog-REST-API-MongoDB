Resources

Post

A Post resource represents a Blog Post.

Attributes:

  • title string -- The title for the Blog Post.
  • subtitle string -- The subtitle for the Blog Post.
  • author string -- The author of the Blog Post.
  • content string -- The content of the Blog Post.
  • date_posted string -- The Date on which Blog Post is posted.

Endpoints:

  • GET /post -- Get all the blog posts.
  • POST /post -- Create a new bog post.
  • GET /post/{string:post_id} -- Get a blog post using unique post_id.
  • PUT /post/{string:post_id} -- Update a blog post using unique post_id.
  • DELETE /post/{string:post_id} -- Delete a blog post using unique post_id.

User

A User resource represents the API user.

Attributes:

  • username string -- The username for the user.
  • password string -- The password for the user.

Endpoints:

  • POST /signup -- Signup a new API user.
  • POST /login -- Login API user, this will provide us with JWT tokens which needs to be used in the Bearer Authorization header for all PUT, POST and DELETE requests for the Post resource.
  • POST /logout -- Logout API user, this will make the JWT tokens provided during login unauthorized to access the PUT and POST endpoints for Post resource.
